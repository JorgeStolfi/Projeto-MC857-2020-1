Hoje fiquei responsável pelo módulo comando_solicitar_pag_cadastrar_usuario.

Esse módulo é responsável por requerir a página de cadastro de usuário, de acordo com a sessão corrente: Se for ativa e de administrador, o formulpario deveria conter um checkbox extra usado para indicar se o novo usuário será ou não administrador; Se não houver sessão ativa ou o usuário não for administrador, o formulário não conterá essa opção.

Esse módulo possui apenas um método que é o "processa", que simplesmente chama a função principal, 'gera', de outro módulo, o 'html_pag_cadastrar_usuario'. Nessa chamada, é usada a sessão recebida como parametro, anulando os demais parametros pedidos por 'html_pag_cadastrar_usuario.gera()' (são eles, 'atibutos' e 'erros'), ignorando também, o parâmetro 'atributos' que recebe.

Então para testar o comando_solicitar_pag_cadastrar_usuario o que fiz, foi:

1. Criar um novo arquivo de teste para o mesmo
2. Alterar os imports para incluí-lo
3. Trocar os módulo e métodos chamados para adequar-se ao "comando_solicitar"
4. Adaptar as variáveis
5. Incluir as demais sessões de teste para 
4. Executar o teste

No final, todas as páginas de teste foram criadas normalmente.

OBS: 
1. Analisando os 3 arquivos (interface, _IMP e _TST) de cada um desses 2 módulos (comando_solicitar_pag_cadastrar_usuario e html_pag_cadastrar_usuario), notei que falta um parâmetro para o teste do html_pag_cadastrar_usuario, pois cada chamada só tem sessão e erros, faltam os atributos.
2. Pelos resultados gerados em teste, mesmo para sessão nula, o formulário ainda apresenta o checkbox de administrador.
