Tarefa: [U] "Meu carrinho" > "Ver" (na primeira poltrona): os campos não editáveis deveriam sair com fundo cinza ou algo assim.

Primeiro acessei o sistema como o usuário "primeiro", na página "Meu carrinho" > "Ver". Constatei que os campos em questão já não são editáveis. Logo a tarefa consiste em apenas mudar o estilo dos campos para ficarem com o fundo cinza.

Olhando na barra do navegador, vi que a página acessava o caminho "/ver_poltrona". Então abri o módulo {comando_ver_poltrona} e, nele, vi que o responsável por gerar o HTML da página é o módulo {html_pag_ver_poltrona}.

Acessando esse segundo módulo, vi que não era ele que gerava o HTML dos campos do formulário em si. Então fui entrando nos módulos recursivamente seguindo o caminho: {html_pag_ver_poltrona} > {html_form_dados_de_poltrona} > {html_form_table} > {html_input}. Sendo esse último o responsável por criar o HTML dos campos do formulário.

Neste módulo, alterei a lógica para adicionar um estilo CSS no trecho HTML do campo que deixa seu fundo cinza caso o parâmetro {editavel} seja falso.

Rodei o teste do módulo e ele gerou a página HTML com os campos não editáveis com fundo cinza.

Como mudei essa lógica no módulo {html_input} e ele é usado para gerar todos os campos de formulário do sistema, um efeito colateral dessa mudança foi que agora todos os campos não editáveis do sistema aparecem com o fundo cinza. Isso, porém, é um comportamento esperado para manter a consistência visual. Não faz sentido um campo não editável ter cores diferentes em páginas diferentes. Assim, a tarefa foi finalizada.
