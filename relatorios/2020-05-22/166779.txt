Para escrever o programa de teste para o módulo html_form_tabela_de_campos, criei um lista de quíntuplas {dados_linhas} com cada rótulo e tipo de cada linha do formulário. Em seguida, para carregar a página já com valores, criei um dicionário {atrs} para popular o formulário, de acordo com o campo {chave} da quíntupla.
Após criar alguns tipos diferentes de entrada de dados, gerei o html a partir de uma requisição de um usuário normal e de um admin. Todos os campos estão visíveis e editáveis para o admin, porém, para o usuário normal, apenas os campos que não são {adm_only} estão visíveis. Nesse caso, eles também são editáveis, porém, não é o comportamento esperado (os campos com {adm_only} deveriam ser visíveis, porém não editáveis para o usuário comum).
